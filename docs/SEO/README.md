# docs/SEO — Documentation SEO moverz.fr

> **Règle Cursor :** Toute action SEO significative (nouvelle page, modification de title/meta, refonte de contenu, changement de structure, audit GSC) **doit être documentée ici**. Créer ou mettre à jour un fichier `.md` correspondant dans ce dossier. Ne jamais laisser une action SEO sans trace écrite.

---

## Convention de nommage

| Préfixe | Usage |
|---------|-------|
| `SEO_JJMMAAAA.md` | Données GSC brutes exportées à une date donnée |
| `SEO_JJMMAAAA_ANALYSE.md` | Analyse + plan d'action issu des données GSC |
| `RECAP-SEO-*.md` | Récapitulatif hebdomadaire des actions effectuées |
| `AUDIT-*.md` | Audit thématique (CTR, LLM, conversion, schemas…) |
| `STRATEGIE-*.md` | Documents de stratégie long terme |
| `PLAN-*.md` | Plans d'action détaillés |
| `OPTIMISATION-*.md` | Optimisations techniques ou contenu |
| `TODO.md` | **Fichier vivant** — plan d'action en cours, mis à jour à chaque analyse |

---

## Workflow SEO recommandé

```
1. Export GSC → SEO_JJMMAAAA.md (données brutes)
2. Analyse    → SEO_JJMMAAAA_ANALYSE.md (diagnostic + plan)
3. Exécution  → mettre à jour TODO.md (cocher les tâches)
4. Récap      → RECAP-SEO-SEMAINE-{date}.md (ce qui a été fait)
```

## Règles pour Cursor

1. **Après chaque analyse GSC** → créer `SEO_JJMMAAAA.md` + `SEO_JJMMAAAA_ANALYSE.md`
2. **Après chaque semaine de travail SEO** → créer `RECAP-SEO-SEMAINE-{date}.md`
3. **Après chaque tâche complétée dans `TODO.md`** → cocher la case `[x]`
4. **Tout nouveau fichier SEO** va dans ce dossier, pas à la racine du projet
5. **Ne jamais modifier** `scoring.md` sans mettre à jour aussi `app/verifications-partenaires/page.tsx` et `app/label-moverz/page.tsx`
